Flashcards & AudioCards — Geração de Questões

Referência para gerar as questões no ChatGPT. Cada tema vira um arquivo que abastece as duas seções (Flashcards e AudioCards — mesmo conteúdo, o AudioCards só acrescenta o player de áudio).

Como usar

Para cada lote, informe ao ChatGPT:

  1. Especialidade — o slug em verde (ex.: cardiologia)
  2. Tema — o nome exato da lista abaixo (ex.: Crise Hipertensiva)
  3. Quantas questões gerar
  4. Um exemplo de pergunta/resposta no estilo desejado
  5. O conteúdo de origem (cole o material)

Prompt para o ChatGPT

Cole tudo abaixo no ChatGPT e preencha os campos no fim.

You are helping build medical study flashcards in Brazilian Portuguese for Revalida exam prep. I will give you (1) a subject, (2) how many cards to create, (3) one example of a good question/answer for style, and (4) source content. You will generate new flashcards FROM the source content, and output them in a strict machine-parseable format that a script will import literally.

WHAT TO GENERATE
- Create flashcards in Brazilian Portuguese only.
- Each card has exactly two parts: a question (Pergunta) and an answer (Resposta).
- STRICT SOURCE FIDELITY: use ONLY facts stated in the source content I paste. Do NOT add outside medical knowledge, do NOT infer facts that aren't there, do NOT include guidance the source doesn't mention. If something isn't in the content, it doesn't become a card.
- Aim for the number of cards I request. If the content genuinely doesn't support that many distinct, accurate cards, make FEWER rather than invent or pad - quality and accuracy over hitting the number.
- Match the STYLE, length, and tone of the example I give you (typically short, direct answers). The example is a style reference only - do NOT include it as a card unless that fact is also in the source content.
- Cover the testable facts in the content. Don't write near-duplicate cards.
- Answers will also be read aloud as audio, so write them as clean, speakable sentences (no abbreviations like "PA" - write "pressao arterial"; spell out numbers naturally where it reads better).

OUTPUT FORMAT (follow EXACTLY - a script parses this literally)
Before the document, on its own line, write the filename:
===FILE: <specialty>__<subject-slug>.md===
where <specialty> is the slug I give you (e.g. cardiologia) and <subject-slug> is the subject lowercased, accents removed, spaces to hyphens (e.g. "Crise Hipertensiva" to crise-hipertensiva).

Then the document, exactly like this:

---
specialty: <specialty-slug>
subject: <Subject Name exactly as I gave it, with normal capitalization and accents>
---

## Card 1
**Pergunta:**
<question text>

**Resposta:**
<answer text>

## Card 2
**Pergunta:**
<question text>

**Resposta:**
<answer text>

(continue, numbered sequentially from 1)

HARD RULES (do not break)
1. Frontmatter at top, delimited by lines of only three dashes (---). Only two keys: specialty, subject.
2. Every card starts with "## Card N" (sequential).
3. Each card has exactly one "**Pergunta:**" line then one "**Resposta:**" line, each on its own line, text on the following line(s).
4. Never use the strings "## Card", "**Pergunta:**", "**Resposta:**", "===FILE:", or a line of only "---" inside the actual question/answer text.
5. No tables, no code fences, no bullet-list-only cards, no commentary before or after, no summary. Output ONLY the ===FILE:=== line and the document.
6. One blank line between cards and between the Pergunta/Resposta blocks.
